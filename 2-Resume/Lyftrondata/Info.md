2. What is the difference between useEffect and useLayoutEffect in React?
useEffect runs after the DOM has been painted, while useLayoutEffect runs synchronously after all DOM mutations but before the browser paints. Use useLayoutEffect only when you need to measure layout or make visual adjustments before the screen is visible.

🧠 3. How does JavaScript handle asynchronous code?
JavaScript uses the event loop, along with Promises, async/await, and callback functions to handle asynchronous operations. Tasks like API calls, file reads, or setTimeout are processed outside the main thread and return results via callbacks or Promises.

🧩 4. What are props and state in React?
Props are read-only data passed from parent to child components. State is local, mutable data managed inside the component using useState or similar hooks. State controls reactivity, while props flow top-down.

I use JWT (JSON Web Tokens) for authentication and role-based access, and input validation with libraries like Joi. I also implement CORS policies, use HTTPS, and sanitize incoming data to prevent attacks like XSS or SQL injection.

🚀 6. What is SSR in Next.js and when would you use it?
SSR (Server-Side Rendering) in Next.js renders the HTML on the server for each request. It’s useful when content changes frequently or when SEO is important. It improves performance for the initial load and helps with search engine indexing.

What are getStaticProps and getServerSideProps?
getStaticProps is used to pre-render a page at build time with static data — ideal for content that doesn't change often.
 getServerSideProps runs on every request and fetches data server-side — great for dynamic, user-specific content.
 Both are used to fetch data before rendering a page.

✅ How is routing handled in Next.js?
Next.js uses file-based routing. Every file in the pages or app directory automatically becomes a route.
 You can create dynamic routes using brackets like [id].js, and nested folders for URL structure.
 For programmatic navigation, you use the next/router API.

✅ What’s the difference between SSR and CSR?
SSR (Server-Side Rendering) fetches and renders the page on the server for every request — good for SEO and real-time data.
 CSR (Client-Side Rendering) loads a minimal HTML and fetches data after the page loads — it's faster for logged-in or dynamic UIs.
 SSR is preloaded; CSR is browser-executed.

What is JWT-based Authentication?
JWT stands for JSON Web Token. It's a stateless, token-based authentication method commonly used in modern web applications.

🔹 How it Works (Simple Flow):
Login:
 User logs in with email/password.


Token Creation:
 Server validates credentials and creates a JWT, which includes user data (like userId, role, etc.) and is signed with a secret.


Token Sent to Client:
 The token is sent back to the client (usually stored in localStorage, sessionStorage, or cookies).


Protected Route Access:
 For every subsequent request to protected APIs, the client sends the JWT in the Authorization header:
 Authorization: Bearer <token>


Verification:
 The server verifies the token using the secret key. If valid, it allows access; if not, it rejects the request.



🔐 Why JWT is Useful
Stateless: No need to store session on the server.


Scalable: Good for distributed systems and microservices.


Secure: Payload is tamper-proof (if properly signed).

This approach is stateless and scalable, and I also implement token expiration and refresh mechanisms for added security."

✅ What’s Inside a JWT Token?
A JWT (JSON Web Token) is a base64-encoded string with three parts:
php-template
CopyEdit
<HEADER>.<PAYLOAD>.<SIGNATURE>

🔹 1. Header (algorithm & token type)
json
CopyEdit
{
  "alg": "HS256",
  "typ": "JWT"
}


🔹 2. Payload (actual data)
This is where you put claims — like user info or roles. For example:
json
CopyEdit
{
  "userId": "12345",
  "email": "user@example.com",
  "role": "admin",
  "iat": 1718000000,  // issued at
  "exp": 1718003600   // expiration (Unix time)
}


🔹 3. Signature
The backend signs the header and payload using a secret key, to prevent tampering:
scss
CopyEdit
HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)


✅ Sample JWT Auth Code in Node.js + Express
js
CopyEdit
const jwt = require("jsonwebtoken");

// Sign a token on login
const generateToken = (user) => {
  return jwt.sign(
    {
      userId: user.id,
      email: user.email,
      role: user.role
    },
    process.env.JWT_SECRET, // secret key
    { expiresIn: "1h" }
  );
};


🔐 Middleware to protect routes
js
CopyEdit
const authenticate = (req, res, next) => {
  const token = req.headers.authorization?.split(" ")[1];

  if (!token) return res.status(401).json({ message: "No token provided" });

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded; // Attach user info to request
    next();
  } catch (err) {
    return res.status(403).json({ message: "Invalid or expired token" });
  }
};
