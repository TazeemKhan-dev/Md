One of the key projects I worked on at Lyftrondata was the Integration Journey — a step-by-step UI workflow designed to help users, especially non-technical ones, connect source systems like MySQL, Oracle, or SQL Server to cloud data warehouses such as Snowflake or Redshift.
The purpose was to simplify data integration and reduce reliance on engineering teams for routine onboarding tasks. The main goal was to streamline the data integration process from a user experience perspective and make onboarding smoother without heavy technical involvement.
The workflow included multiple stages: selecting the target warehouse, choosing the data source, entering connection credentials, viewing fetched schemas, selecting specific tables or views, and mapping them to the target warehouse.
From a frontend standpoint, I was responsible for:
Designing and implementing a clean, responsive, and accessible UI using React.js and Material UI.


Managing multi-step UI flow using Redux Toolkit for global state consistency, with form validation handled in the credential input step.


Ensuring smooth interaction and feedback to guide users confidently through each integration step.


Building reusable UI components to maintain consistency across the workflow and improve development efficiency.


The platform as a whole was used by data teams and enterprise clients to centralize, manage, and transform large volumes of data for analytics and reporting. The Integration Journey played a major role in improving onboarding and expanding the platform’s usability to non-developers.
This project not only improved onboarding time but also made the platform more accessible to business users. It taught me how to balance frontend complexity with usability, build scalable component-based UIs, and focus on delivering an intuitive experience from the user's perspective.
That’s a quick overview of the project from a frontend lens — happy to dive deeper into any part if needed.
